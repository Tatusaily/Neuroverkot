# I do not use np.random.shuffle(), because then I would have to convert the dataset to a numpy array
# and then back to a pandas dataframe, which is not very efficient
dataset = dataset.sample(frac=1).reset_index(drop=True)

# one-hot encode the class labels
dataset['class'] = pd.Categorical(dataset['class']).codes

# split the dataset into input and output features
X = pd.DataFrame(dataset.drop('class', axis=1))
Y = to_categorical(dataset['class'])

Actual modeling with Keras

  1. Initialize the model with input layer of 4 features
  2. Create a hidden layer with 32 neurons with a non-linear ReLU activation function
  3. Final output layer with 3 neurons (3 Iris classifications)

Optimizer is the Adam optimizer. I started with the RMSProp, but Adam gave better results.

Summary of the model's architechture and training is printed and graphed below.

In[36]:
# start with a model so we can do a summary
model = Sequential([
    Input(shape=(4,)),
    Dense(32, activation='relu'),
    Dense(3, activation='softmax')
])
model.summary()

# Loop for testing different batch sizes
# I will use the same model and settings for all batch sizes
# Loop includes a plot of the loss and accuracy for each batch size,
# as well as the final accuracy and loss and the number of epochs needed to reach it

for batch_size in [4, 8, 16, 32]:
    # Set up the model
    # If model is outside the loop only, the model does not reset properly
    model = Sequential([
        Input(shape=(4,)),
        Dense(32, activation='relu'),
        Dense(3, activation='softmax')
    ])

    optimizer = keras.optimizers.Adam(
        learning_rate=0.0005
    )

    model.compile(optimizer=optimizer,
                loss='categorical_crossentropy',
                metrics=['accuracy'])

    # model.summary()

    early_stopping = keras.callbacks.EarlyStopping(
        monitor='val_loss',
        patience=10,
        min_delta=0.0005,
        restore_best_weights=True
    )

    # Start the training
    print(f"Training with batch size: {batch_size}")
    model.fit(X, Y, 
              epochs=1000, 
              batch_size=batch_size, 
              validation_split=0.2,
              verbose=0,
              callbacks=[early_stopping])
    # report the final accuracy and epoch number
    loss, accuracy = model.evaluate(X, Y, verbose=0)
    print(f"Final accuracy: {accuracy:.2f}")
    print(f"Final loss: {loss:.2f}")
    print(f"Number of epochs: {early_stopping.stopped_epoch if early_stopping.stopped_epoch > 0 else 1000}")
    print()

    # plot the loss and accuracy
    # Extract training history
    history = model.history.history
    plt.figure(figsize=(12, 5))

    # Plot accuracy metrics
    plt.subplot(1, 2, 1)
    plt.plot(history['accuracy'], label='Training Accuracy')
    plt.plot(history['val_accuracy'], label='Validation Accuracy')
    plt.title('Model Accuracy')
    plt.ylabel('Accuracy')
    plt.xlabel('Epoch')
    plt.legend(loc='lower right')

    # Plot loss metrics
    plt.subplot(1, 2, 2)
    plt.plot(history['loss'], label='Training Loss')
    plt.plot(history['val_loss'], label='Validation Loss')
    plt.title('Model Loss')
    plt.ylabel('Loss')
    plt.xlabel('Epoch')
    plt.legend(loc='upper right')

    plt.tight_layout()
    plt.show()

Analysis

Model very quickly achieves accuracy greater than 0.95 in all batch sizes.

Size 16 and 32 models were slower to find good accuracy and loss. Size 16 also has a gap between Training and Validation accuracy.

Sizes 4 and 8 reach good accuracy much faster, and also reach the early stop much faster 460 vs 690 epochs.

It is difficult to say which of the smaller batch size models trained better. 4 has marginally better accuracy and final loss.
